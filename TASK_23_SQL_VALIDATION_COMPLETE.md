## Problem Identified

From the context transfer, Groq was generating invalid SQL Server syntax with three recurring patterns:

### Pattern 1: Bare FROM Inside Subqueries (MOST COMMON)
```sql
-- WRONG (Groq generates):
SELECT COUNT(DISTINCT Object) AS unique_objects,
       AVG(modification_count) AS average_modifications
FROM (
    FROM DatabaseLog
    (Object, COUNT(*) AS modification_count)
    FROM DatabaseLog
    GROUP BY Object
    WHERE Object IS NOT NULL
) AS modifications

-- CORRECT:
SELECT COUNT(DISTINCT Object) AS unique_objects,
       AVG(1.0 * modification_count) AS average_modifications
FROM (
    SELECT Object, COUNT(*) AS modification_count
    FROM DatabaseLog
    WHERE Object IS NOT NULL
    GROUP BY Object
) t
```

### Pattern 2: Floating Column Lists Before FROM
```sql
-- WRONG:
(Object, COUNT(*) AS modification_count) FROM DatabaseLog

-- CORRECT:
SELECT Object, COUNT(*) AS modification_count FROM DatabaseLog
```

### Pattern 3: Incomplete UNION ALL Statements
```sql
-- WRONG:
SELECT col1 FROM table1 UNION ALL

-- CORRECT:
SELECT col1 FROM table1
```

### 2. New Validation Methods

#### Method 1: `_fix_bare_from_in_subquery(sql: str) -> str`

**Purpose**: Fix bare FROM inside subqueries (most common Groq error)

**Algorithm**:
1. Detect pattern: `FROM ( FROM`
2. Find matching closing parenthesis using paren counter
3. Extract subquery content between parens
4. Remove leading `FROM` keyword if present
5. Prepend `SELECT *` if subquery doesn't start with SELECT
6. Reconstruct SQL with fixed subquery
7. Log the fix for debugging

**Example**:
```python
# Input:
FROM (FROM DatabaseLog (Object, COUNT(*)) GROUP BY Object) t

# Output:
FROM (SELECT * FROM DatabaseLog WHERE Object IS NOT NULL GROUP BY Object) t
```

#### Method 2: `_fix_floating_column_list(sql: str) -> str`

**Purpose**: Fix floating column lists before FROM

**Algorithm**:
1. Find pattern: `(col1, col2, ...) FROM`
2. Verify it's a column list (contains only word chars and commas)
3. Prepend `SELECT` keyword
4. Process matches in reverse to maintain string indices
5. Log each fix

**Example**:
```python
# Input:
(Object, COUNT(*)) FROM DatabaseLog

# Output:
SELECT Object, COUNT(*) FROM DatabaseLog
```

#### Method 3: `_fix_incomplete_union_all(sql: str) -> str`

**Purpose**: Fix incomplete UNION ALL statements

**Algorithm**:
1. Count SELECT keywords in SQL
2. Count UNION ALL keywords in SQL
3. If UNION ALL count >= SELECT count, something is wrong
4. Remove trailing UNION ALL if incomplete
5. Log the fix

**Example**:
```python
# Input:
SELECT col1 FROM table1 UNION ALL

# Output:
SELECT col1 FROM table1
```

```ini
[dialect]
name = SQL Server
prompt_snippet = 
    ⚠️  CRITICAL SUBQUERY RULES (MOST IMPORTANT):
    - NEVER write bare FROM inside subqueries
    - NEVER write floating column lists before FROM
    - ALWAYS use complete SELECT ... FROM ... WHERE ... GROUP BY structure
    - Example CORRECT: SELECT col1, COUNT(*) FROM (SELECT col1 FROM table WHERE ...) t GROUP BY col1
    - Example WRONG: FROM (FROM table (col1, COUNT(*)) GROUP BY col1)
    
    SYNTAX RULES:
    - Use TOP N for limiting rows: SELECT TOP 10 ...
    - Never use LIMIT.
    - For strings: VARCHAR(8000) or VARCHAR(MAX) — never VARCHAR without length.
    - Aggregates (SUM, AVG) require numeric types — CAST to DECIMAL(18,2) or FLOAT if needed.
    - For AVG of INT columns, use AVG(1.0 * column_name) to force decimal division (e.g., 3.45 not 3).
    - Dates: DATEADD, DATEDIFF, CONVERT(date, ...).
    - No Snowflake functions (QUALIFY, LISTAGG, ARRAY_AGG).
    
    UNION ALL RULES:
    - Both sides must be complete SELECT statements with identical column counts
    - Example: SELECT col1, col2 FROM table1 UNION ALL SELECT col1, col2 FROM table2
    
    CTE RULES:
    - Use CTEs (WITH clause) for complex multi-step queries — cleaner than nested subqueries
    - Example: WITH cte AS (SELECT ...) SELECT * FROM cte
    
    TABLE RULES:
    - Do NOT invent tables like 'customers', 'orders', 'revenue', 'sales'
    - Use ONLY real schema tables listed in the LIVE SCHEMA section
```

```sql
    GROUP BY Object
    WHERE Object IS NOT NULL
```

**Validation Process**:
