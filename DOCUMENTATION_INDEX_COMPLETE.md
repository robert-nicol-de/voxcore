## Key Concepts

### Three-Line Architecture

**Line 1**: Platform-specific system prompt (pre-LLM)
- LLM receives platform-specific rules BEFORE generating SQL
- Reduces hallucinations, improves accuracy
- File: `backend/voxquery/core/sql_generator.py`

**Line 2**: SQL validation & rewrite (post-LLM)
- SQL is validated and rewritten to platform-compliant syntax
- Catches errors, rewrites syntax, falls back to safe query
- File: `backend/voxquery/core/engine.py`

**Line 3**: Execute final_sql (never raw LLM output)
- Only validated, platform-compliant SQL is executed
- Prevents SQL injection, ensures platform compatibility
- File: `backend/voxquery/api/query.py`

### Platform Isolation

Each platform is completely isolated:
- SQL Server login → only `sqlserver.ini` loaded
- Snowflake login → only `snowflake.ini` loaded
- Zero cross-contamination

### Defense in Depth (4 Layers)

1. **System Prompt** - LLM sees rules first
2. **Forbidden Keyword Detection** - Catches mistakes
3. **Forbidden Table Detection** - Whitelist enforcement
4. **Runtime Rewrite** - Syntax cleanup
