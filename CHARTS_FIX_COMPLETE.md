## Problem

Charts were not displaying in the frontend even though they were being generated by the backend.

## Root Cause

The API response in `backend/voxquery/api/query.py` was not including all the required fields from the engine response:
- `query_type` - was missing
- `confidence` - was missing
- `explanation` - was missing
- `tables_used` - was missing

These fields are needed by the frontend to properly display the response and validate the charts.

## Solution

Updated the `QueryResponse` in `backend/voxquery/api/query.py` to include all fields from the engine result:

```python
return QueryResponse(
    question=request.question,
    sql=result.get("sql"),
    query_type=result.get("query_type"),           # ← ADDED
    confidence=result.get("confidence"),           # ← ADDED
    explanation=result.get("explanation"),         # ← ADDED
    tables_used=result.get("tables_used"),         # ← ADDED
    data=result.get("data"),
    row_count=result.get("row_count", 0),
    execution_time_ms=result.get("execution_time_ms", 0.0),
    error=result.get("error"),
    chart=charts.get("bar") if charts else None,
    charts=charts if charts else None,
)
```
